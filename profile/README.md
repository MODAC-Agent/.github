### 🎯 목표

- AI의 한계와 강점을 직접 실험을 통해 이해한다
- 실제 문제를 만났을 때 적절한 LLM 활용 방식을 판단하고 적용할 수 있는 역량을 기른다
- 각 접근법의 패턴과 판단 기준을 정리한 리포트를 완성한다

<br>

### 🗓 주차별 학습 계획

| 주차 | 주제 | 내용 |
|:---:|---|---|
| 1주 | 🔍 문제 정의 | 해결할 문제를 정의하고, 단일 프롬프트나 도구를 더해 간단히 문제 해결을 시도합니다. |
| 2주 | 🛠 활용 방식 설계 | 문제 해결에 필요한 도구, 에이전트의 구조를 설계하고 구체화합니다. |
| 3주 | 🚀 1차 적용 & 사례 공유 | 실제 문제 해결을 시도하고, 잘 된 점과 어려웠던 점을 사례 중심으로 공유합니다. |
| 4주 | 🔧 활용 방식 개선 | 3주차에서 발견한 문제를 해결하고, 에이전트의 일관성과 품질을 높입니다. |
| 5주 | 📝 패턴 정리 & 리포트 작성 | 전체 사례를 바탕으로 유효한 활용 패턴과 판단 기준을 정리하고, 리포트 형태로 마무리합니다. |

<br>

### 📂 레포지토리 구조

```
├── week1/          # 문제 정의 & 초기 실험
├── week2/          # 활용 방식 설계
├── week3/          # 1차 적용 사례
├── week4/          # 개선 내용
├── week5/          # 패턴 정리 & 최종 리포트
└── report/         # 최종 결과 리포트
```

<br>

### 🙌 참여 방법

매주 본인의 실험 내용을 해당 주차 폴더에 추가하고, PR을 통해 공유합니다.

1. **토요일 자정까지** — 학습 내용을 정리해 PR로 리뷰 요청
2. **화요일 모임 전까지** — 팀원 PR 리뷰 및 피드백 반영
3. **화요일 모임** — 전체 회고 및 의견 나누기

<br>

### 📋 기여 규칙

- 실험 내용은 `이름_주제.md` 형태로 작성합니다.
- 코드가 있을 경우 실행 방법(README 또는 주석)을 함께 남겨주세요.
- 잘 안 된 사례도 적극적으로 공유해주세요. 실패 사례가 가장 좋은 학습 자료입니다. 🙂
